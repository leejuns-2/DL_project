# Foundation Model 기반 에너지 리포트·뉴스 신호 분석 보고서

## 1. 프로젝트 한눈에 보기

이 프로젝트는 에너지·기후 관련 **뉴스**와 **PDF 리포트**를 Foundation Model로 읽고, 그 내용을 숫자 신호로 바꾼 뒤 실제 에너지 ETF·기업 주가와 비교한 분석이다.

쉽게 말하면:

> AI가 뉴스 제목과 긴 PDF 보고서를 읽는다.  
> 그 안에 재생에너지 기회, 화석연료 압력, 전력망 투자, 기후 리스크가 얼마나 강하게 담겨 있는지 점수로 만든다.  
> 마지막으로 그 점수가 실제 주가 움직임과 어느 정도 연결되는지 확인한다.

핵심 결과:

| 구분 | 결론 |
|---|---|
| Foundation Model 활용 | 뉴스 감성 분석과 PDF 의미 분석에 적용 완료 |
| Few-shot learning | PDF 문단을 사람이 만든 예시 문장과 비교해 점수화 |
| Downstream task | 텍스트 신호를 ETF·기업 주가 수익률과 연결 |
| 예측 성능 | 강한 주가 예측 모델이라고 말하기는 어려움 |
| 발표 포인트 | “비정형 텍스트를 시장 분석용 수치 신호로 바꾸는 파이프라인”으로 설명하는 것이 가장 적절 |

## 2. 전체 파이프라인

```text
[뉴스 데이터]
GDELT 뉴스 헤드라인
-> FinBERT 감성 분류
-> 주간 뉴스 감성 신호 생성

[PDF 리포트]
IEA / ExxonMobil / Eaton 보고서 PDF
-> 텍스트 추출
-> 관련 문단 검색
-> Foundation Model 임베딩
-> Few-shot 점수화
-> 리포트 신호 생성

[시장 데이터]
yfinance 주가 데이터
-> 주간 수익률 계산
-> ICLN - XLE 스프레드 계산

[최종 분석]
뉴스 신호 + PDF 리포트 신호 + 기후 지표 + 주가 수익률
-> 시차 상관 분석
-> 이동 상관 분석
-> 시각화 및 해석
```

이 흐름에서 중요한 점은 PDF를 단순 요약에서 끝내지 않았다는 것이다. PDF 내용을 숫자로 바꾸고, 그 숫자를 주가 데이터와 연결했기 때문에 Foundation Model의 출력이 downstream task에 사용되었다.

---

## 3. 사용한 Foundation Model

| 모델 | 사용 위치 | 쉬운 설명 |
|---|---|---|
| `ProsusAI/finbert` | 뉴스 헤드라인 감성 분석 | 뉴스 제목이 금융시장 관점에서 긍정적인지, 부정적인지, 중립인지 판단 |
| `sentence-transformers/all-MiniLM-L6-v2` | PDF 문단 의미 분석 | 보고서 문단과 예시 문장이 의미상 얼마나 가까운지 계산 |

Few-shot learning은 다음 방식으로 적용했다.

1. `재생에너지 기회`, `화석연료 압력`, `전력망/전기화`, `기후 리스크`에 해당하는 예시 문장을 몇 개 만든다.
2. PDF에서 관련 문단을 찾는다.
3. Foundation Model로 예시 문장과 PDF 문단을 벡터로 바꾼다.
4. 의미가 가까울수록 해당 주제의 점수를 높게 준다.

즉, 모델 전체를 새로 학습한 것은 아니지만, 적은 예시를 기준으로 새로운 문서를 분류·점수화하는 few-shot 방식의 downstream task를 구현했다.

## 4. 데이터 구성

| 데이터 | 내용 | 현재 상태 |
|---|---|---|
| 주가 데이터 | ICLN, XLE, NEE, XOM, ETN | 312주, 결측 없음 |
| 기후 데이터 | NASA POWER 기반 TAI | 311주, 결측 없음 |
| 뉴스 데이터 | GDELT 에너지·기후 뉴스 | 3,340건 |
| 뉴스 감성 데이터 | FinBERT로 만든 주간 감성 신호 | 100개 관측 주차 |
| PDF 리포트 | IEA 3개, ExxonMobil 1개, Eaton 1개 | 총 5개 보고서 |

주의할 점:

- 뉴스 데이터는 GDELT 제한 때문에 모든 주차를 완벽하게 덮지는 못한다.
- PDF 리포트는 5개만 사용했기 때문에 통계 검증보다는 파이프라인 시연 성격이 강하다.
- 그래도 “PDF -> Text -> Summary”보다 발전된 “PDF -> Text -> Evidence Retrieval -> Embedding -> Few-shot Scoring -> Market Signal” 구조를 만들었다.

## 6. PDF 리포트 신호와 이후 4주 수익률

| 보고서 | 전환 신호 | 4주 후 ET_SPREAD | 4주 후 ICLN | 4주 후 XLE | 4주 후 ETN |
|---|---:|---:|---:|---:|---:|
| ExxonMobil ACS 2023 | 1.321 | -0.082 | -0.054 | 0.028 | -0.025 |
| IEA World Energy Outlook 2023 | 1.438 | 0.121 | 0.052 | -0.062 | 0.179 |
| IEA Oil and Gas Net Zero 2023 | 1.345 | 0.100 | 0.094 | -0.005 | 0.042 |
| IEA Renewables 2023 | 1.453 | -0.044 | -0.055 | -0.014 | 0.150 |
| Eaton Annual Report 2023 | 1.373 | -0.080 | 0.001 | 0.084 | 0.115 |

### 쉬운 해석

- `ET_SPREAD`는 `ICLN 수익률 - XLE 수익률`이다.
- 이 값이 양수면 재생에너지 쪽이 화석연료 쪽보다 상대적으로 더 좋았다는 뜻이다.
- 이 값이 음수면 화석연료 쪽이 재생에너지 쪽보다 상대적으로 더 좋았다는 뜻이다.

보고서별로 보면:

- `IEA World Energy Outlook 2023` 이후 4주 동안 `ET_SPREAD`가 `0.121`이었다. 이 기간에는 재생에너지 쪽이 석유·가스 쪽보다 상대적으로 좋았다.
- `IEA Oil and Gas Net Zero 2023` 이후에도 `ET_SPREAD`가 `0.100`으로 양수였다.
- 반대로 `IEA Renewables 2023` 이후에는 전환 신호가 가장 높았지만 `ET_SPREAD`는 음수였다. 즉, 보고서 내용이 항상 바로 주가 방향과 일치하지는 않았다.
- `ETN`은 여러 보고서 이후 비교적 양호한 수익률을 보였다. 전력망·전기화 테마와 연결해 추가 분석할 만하다.

최종 해석:

> PDF 신호와 주가를 연결하는 데는 성공했다.  
> 다만 보고서 수가 5개라서 “통계적으로 유의한 예측 모델”이라고 주장하면 안 된다.  
> 현재 단계에서는 Foundation Model 기반 신호 생성과 downstream 연결을 보여주는 결과로 보는 것이 맞다.

## 7. 뉴스 감성 기반 시차 분석 결과

| 가설 | 결과 | 쉬운 해석 |
|---|---|---|
| H1: 기후 이상 지수가 뉴스 감성보다 먼저 움직인다 | 지지되지 않음 | 기후 지표가 높아진 뒤 뉴스 감성이 일관되게 따라 움직인다고 보기 어렵다 |
| H2: 긍정적 뉴스 감성이 재생에너지 상대 수익률을 높인다 | 지지되지 않음 | 뉴스 감성이 좋아졌다고 해서 ICLN이 XLE보다 좋아지는 패턴은 뚜렷하지 않았다 |
| H3: 특정 이벤트 기간에는 관계가 강해진다 | 부분 관찰 | 2022년 하반기와 2023년 초에는 관계가 더 강하게 보이는 구간이 있었다 |

숫자로 보면:

| 관계 | Best lag | r | p-value | 쉬운 해석 |
|---|---:|---:|---:|---|
| TAI -> NSS_ADJ | +2 | -0.121 | 0.236 | 관계가 약하고 통계적으로도 뚜렷하지 않음 |
| NSS_ADJ -> ET_SPREAD | +4 | -0.189 | 0.062 | 약한 음의 관계가 있지만 일반 기준에서는 유의하다고 말하기 어려움 |
| NSS_ADJ -> NEE | +2 | -0.181 | 0.073 | 약한 관계는 보이나 확실하지 않음 |
| NSS_ADJ -> XOM | +4 | 0.216 | 0.033 | 약하지만 통계적으로 관찰되는 관계 |
| NSS_ADJ -> ETN | +3 | 0.200 | 0.048 | 약하지만 통계적으로 관찰되는 관계 |

상관계수 `r`을 쉽게 읽는 법:

- `r`이 1에 가까우면 같이 움직이는 경향이 강하다.
- `r`이 -1에 가까우면 반대로 움직이는 경향이 강하다.
- `r`이 0에 가까우면 뚜렷한 관계가 약하다.
- 이 프로젝트의 `r` 값들은 대부분 `0.2` 안팎이라 강한 관계는 아니다.

`p-value`를 쉽게 읽는 법:

- 보통 `0.05`보다 작으면 통계적으로 관찰할 만하다고 본다.
- `XOM`, `ETN`은 이 기준을 일부 만족했다.
- 하지만 값이 작더라도 상관계수가 약하므로 과장하면 안 된다.

## 9. 최종 결론

이 프로젝트의 결론은 “주가를 정확히 예측했다”가 아니다.

더 정확한 결론은 다음이다.

> Foundation Model을 이용해 뉴스와 PDF 리포트라는 비정형 텍스트를 시장 분석용 수치 신호로 변환했고, 그 신호를 에너지 ETF 및 기업 주가 수익률과 연결하는 downstream 분석 파이프라인을 구현했다.

발표에서 강조하면 좋은 점:

1. 교수님이 예시로 든 `PDF -> Text -> Summary` 흐름을 그대로 멈추지 않고 확장했다.
2. PDF 문단 검색, 임베딩, few-shot 점수화, 주가 연결까지 추가했다.
3. Foundation Model을 단순 호출이 아니라 분석 파이프라인의 중간 엔진으로 사용했다.
4. 결과를 과장하지 않고, 신호 생성과 분석 가능성을 중심으로 해석했다.

## 10. 한계와 보완 방향

| 한계 | 의미 | 보완 방법 |
|---|---|---|
| PDF 보고서 수가 5개 | 통계 결론을 내리기에는 표본이 적음 | 보고서를 연도별·기업별로 더 추가 |
| 뉴스 데이터가 일부 주차만 존재 | 모든 주차를 완전히 비교하기 어려움 | 수집 기간을 넓히거나 다른 뉴스 API 추가 |
| PDF 요약이 추출형 중심 | 생성형 요약보다 문장이 딱딱할 수 있음 | LLM 요약 모델 추가 |
| few-shot 예시가 사람이 만든 기준에 의존 | 예시 문장에 따라 점수가 달라질 수 있음 | 예시 문장 다양화 및 민감도 분석 |
| 상관관계 분석 중심 | 인과관계를 말할 수 없음 | 회귀분석, 이벤트 스터디, 예측 검증 추가 |

## 11. 다른 PDF 일반화 검증

2023년 리포트만으로는 모델이 특정 연도 자료에만 맞춰진 것인지 알기 어렵다. 그래서 2023년이 아닌 PDF 3개를 추가로 넣어 검증했다.

검증 자료:

| 검증 PDF | 기대 라벨 | 모델 예측 | 결과 |
|---|---|---|---|
| IEA World Energy Outlook 2022 | ICLN/NEE | ICLN/NEE | 일치 |
| IEA Renewables 2022 | ICLN/NEE | ICLN/NEE | 일치 |
| IEA Electricity 2024 | ETN | ETN | 일치 |

검증 정확도:

> 3개 중 3개 일치, `1.000`

해석:

- 이 결과는 2023년 자료가 아닌 PDF에서도 방향성이 맞게 나온다는 1차 근거다.
- 다만 검증 자료가 3개뿐이므로 “모든 PDF에서 정확하다”고 말하면 안 된다.
- 현재는 MVP 수준의 일반화 점검으로 보는 것이 맞다.
- 더 엄밀한 검증을 하려면 연도, 기관, 기업을 넓혀 최소 20개 이상의 PDF 검증 세트를 만들어야 한다.

추가로 보완한 부분:

- `IEA Electricity 2024`가 처음에는 재생에너지 신호와 전력망 신호를 혼동했다.
- 원인은 전력·전기화 few-shot 예시가 부족했기 때문이다.
- 전력 수요, 송배전망, 전력 시스템 유연성, 데이터센터 전력 수요 예시를 추가한 뒤 기대 라벨과 일치했다.

| 파이프라인 | PDF·뉴스 수집부터 Foundation Model, few-shot 점수화, 시장 연결까지의 흐름 |
| PDF 신호 | 보고서별 재생에너지, 화석연료 압력, 전력망, 기후 리스크 점수 |
| 시장 연결 | 리포트 신호와 이후 4주 수익률 비교 |
| 뉴스 분석 | 뉴스 감성 신호와 주가·기업 수익률 관계 |
| 산출 이미지 | Figure 1~5를 대시보드 안에서 바로 확인 |
| 산출물 링크 | 결과 보고서, 요약표, 가이드라인으로 이동 |

쉽게 말하면:

> `project_report.md`는 제출용 보고서이고,  
> `mvp_dashboard.html`은 발표나 시연 때 바로 보여줄 수 있는 결과 화면이다.

MVP의 역할:

1. 프로젝트를 처음 보는 사람이 핵심 결과를 빠르게 이해할 수 있다.
2. Foundation Model 파이프라인이 실제 산출물로 이어졌다는 점을 보여준다.
3. 보고서, 표, 그림, 해석을 하나의 화면에서 연결한다.
4. 추후에는 사용자가 PDF를 업로드하면 자동으로 신호를 계산하는 앱으로 확장할 수 있다.

확장 아이디어:

| 현재 MVP | 다음 단계 앱 |
|---|---|
| 정적 HTML 대시보드 | PDF 업로드 기능 |
| 기존 분석 결과 표시 | 업로드한 PDF 실시간 점수화 |
| 이미지 기반 시각화 | Plotly 기반 인터랙티브 차트 |
| 고정 데이터 사용 | 기업·ETF 선택 기능 |
| 결과 해석 텍스트 제공 | LLM 기반 자동 해석 생성 |

## 13. PDF 업로드 앱

정적 HTML MVP에서 한 단계 더 나아가, 사용자가 PDF를 직접 업로드하면 실제 파이프라인이 실행되는 Streamlit 앱을 추가했다.

실행 명령:

```bash
streamlit run app_pdf_mvp.py
```

앱 작동 흐름:

```text
사용자 PDF 업로드
-> PDF 텍스트 추출
-> 에너지 전환 관련 문단 검색
-> Foundation Model 임베딩
-> Few-shot 주제 점수 계산
-> 쉬운 요약 및 근거 문단 표시
-> 보고서 날짜 이후 4주 수익률 연결
```

앱에서 확인할 수 있는 결과:

| 항목 | 의미 |
|---|---|
| 핵심 점수 | 업로드한 PDF가 재생에너지, 화석연료 압력, 전력망, 기후 리스크 중 어디에 가까운지 |
| 자산 힌트 | PDF 내용이 ICLN/NEE, XLE/XOM, ETN 중 어떤 시장 테마와 가까운지 |
| 근거 문단 | 점수를 만든 데 사용된 실제 PDF 문단 |
| 4주 수익률 | 보고서 날짜 이후 ETF·기업 수익률과의 연결 |
| 처리 현황 | 추출 페이지 수, 분석 문단 수, 근거 문단 수 |

이 앱의 의미:

> 이제 프로젝트는 “분석 결과를 보여주는 보고서”에서 끝나지 않고,  
> 새로운 PDF를 넣으면 Foundation Model 기반 분석이 실제로 작동하는 MVP가 되었다.
