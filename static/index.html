프로젝트 해석 기준

무엇을 만들었나

뉴스는 시장 분위기 시계열 신호로, PDF는 리포트 기반 이벤트 신호로 변환해 주가 데이터와 연결하는 파이프라인입니다.

모델 활용 범위

MiniLM은 사전학습 Transformer 임베딩 모델이며, MiniLM 본체는 고정하고 소수 라벨 예시로 downstream 분류 헤드를 학습합니다. Gemini는 근거 문단 요약에 사용합니다.

주의할 해석

수익률 연결과 포트폴리오 계산은 과거 수익률 기반 시나리오이며, 미래 예측·투자 추천·인과관계 증명이 아닙니다.

방법론 정정 및 논리적 주의

항목잘못 말하면 위험한 표현이 프로젝트의 정확한 표현
Few-shot 대형 foundation model 전체를 fine-tuning했다 MiniLM 임베딩은 고정하고, 소수 라벨 예시로 Logistic Regression 분류 헤드를 few-shot 방식으로 학습
MiniLM 대형 생성형 LLM을 사용했다 사전학습 Transformer 기반 범용 문장 임베딩 모델을 사용했다
검증 소규모 일치 결과를 정량 정확도로 일반화했다 소규모 일반화 점검에서 15개 PDF가 기대 방향과 일치했다. 다만 정량 일반화 성능으로 과장하지 않는다
뉴스·PDF 연결 뉴스 분석과 PDF 앱이 따로 존재한다 뉴스는 시장 분위기 신호, PDF는 리포트 이벤트 신호, 주가는 downstream 연결 대상
Few-shot 검증 테스트 셋에서 정확도를 측정했다 별도의 held-out test set이 없음. 15개 검증 PDF는 방향 일치 여부만 확인하는 MVP 탐색 수준이며, 정량 일반화 성능 평가가 아님
점수 정규화 각 테마 점수가 절대 확률이다 분류기 출력 확률을 per-report min-max 정규화하여 상대적 순위로 변환. 보고서 간 점수 직접 비교 불가
